## What changed
This PR rolls the WatchParty release work into one clean branch so the app ships as a single coherent update instead of a stack of conflicting branches. It keeps the lazy PartyKit room initialization already merged on `main`, applies the bright party-themed visual system across the app shell and room flows, adds Lucide icons in the key interaction points, and brings in the generated favicon/social image assets with matching metadata.

## Why
The repo had multiple overlapping branches touching the same landing page, room page, metadata, and shared UI components. That is exactly how you end up with half-merged styling and broken release context. This PR consolidates the party-vibes design pass, the favicon/OG asset work, and the final icon pass into one reviewable release branch while preserving the lazy-init backend fix as the source of truth.
